## Objetivo

Garantir que a documentação publicada corresponde ao código, definir a
propriedade (ownership) dos arrays de dados e validar entrada numérica. Encerra
a Fase 1 sem aumentar escopo funcional.

## Escopo desta versão

### README compilável no CI (§4.7 · §15.8)

- [ ] Todos os exemplos do README compilam.
- [ ] README não contém propriedades inexistentes.
- [ ] Mudança de API que quebra o exemplo quebra o CI.
- [ ] Exemplos de streaming e sincronização possuem teste.

### Ownership e mutabilidade dos arrays (§4.8 · §15.9)

- [ ] Padrão documentado: `copy` (padrão seguro) vs `borrowed` (imutável).
- [ ] Arrays copiados não mudam após mutação externa.
- [ ] Modo `borrowed` marcado explicitamente como imutável.
- [ ] Extensões e caches permanecem consistentes.

### Validação de dados numéricos e gaps (§4.9 · §15.10)

- [ ] Comprimentos incompatíveis são rejeitados.
- [ ] X não monotônico é detectado; `Infinity` é rejeitado.
- [ ] `NaN` em Y segue regra documentada (base para gaps na v1.6.0).
- [ ] Arrays vazios não quebram escalas.
- [ ] Erros indicam série e posição do valor inválido.
