# v1.1.0 — Correções críticas (P0, parte 1)

> **Origem:** improvements §4.1, §4.3, §4.4 · §15 passos 1–3 · roadmap Fase 1
> **Semver:** minor. Contém `behavior fix` (ver nota no fim).
> **Risco:** baixo.

## Objetivo

Remover comportamentos incorretos de domínio, teclado e animação, sem adicionar
escopo funcional. Base de confiabilidade antes de crescer a API.

## Escopo desta versão

### Corrigir `yMin` / `yMax` (§4.1 · §15.1)

- [ ] `yMin` e `yMax` só usam domínio automático quando o limite é `undefined`.
- [ ] `yMin: 0` e `yMax: 0` são preservados (falsy não pode virar "auto").
- [ ] Intervalos negativos funcionam.
- [ ] Configuração inválida gera erro descritivo em desenvolvimento.

### Navegação por teclado (§4.3 · §15.2)

- [ ] Setas navegam por ponto; `Shift + seta` por dez pontos.
- [ ] Cursor nunca sai do intervalo; foco inicial em ponto válido.
- [ ] Série oculta não é usada como referência.
- [ ] Valor atual anunciado por tecnologia assistiva.

### `prefers-reduced-motion` (§4.4 · §15.3)

- [ ] Streaming continua atualizando com reduced-motion ativo.
- [ ] Sem animação quando a preferência está ativa; `invalidate()` ainda desenha.
- [ ] Modo é reavaliado se a preferência mudar em runtime (`matchMedia`).

## Arquivos provavelmente afetados

- `src/charts/chart-base.ts` — resolução de domínio Y, loop de teclado, scheduler.
- `src/types.ts` — contrato de `yMin`/`yMax` (aceitar `undefined` explícito).
- `src/render/crosshair.ts` — cursor de teclado / referência de série.

## Critérios de aceite

- [ ] `yMin: 0` é preservado após `setData`; `yMax: 0` idem.
- [ ] Intervalos negativos funcionam.
- [ ] Domínio automático só quando o limite é `undefined`.
- [ ] Configurações inválidas geram erro descritivo em desenvolvimento.
- [ ] Testes unitários para zero, negativos e intervalo inválido.
- [ ] Setas navegam por ponto; `Shift + seta` por dez pontos.
- [ ] Cursor nunca fica fora do intervalo; foco inicial válido.
- [ ] Série oculta não é usada como referência.
- [ ] Valor atual anunciado por tecnologia assistiva; documentação corresponde ao comportamento.
- [ ] Streaming continua atualizando com reduced-motion; sem animação quando ativo.
- [ ] `invalidate()` causa desenho; modo atualiza se a preferência mudar em runtime.
- [ ] Teste de integração com `matchMedia`.

## Nota de CHANGELOG / semver

`minor` com **behavior fix**:

> `yMin`/`yMax` agora respeitam `0` e só caem no domínio automático quando
> `undefined`. Código que dependia do descarte de `0` precisa passar `undefined`
> explicitamente. Documentar em "Behavior fixes".
