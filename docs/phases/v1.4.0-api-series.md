# v1.4.0 — API de séries e operações em lote (P1, estabilização A)

> **Origem:** improvements §5.1, §5.2, §5.3 · §15 passos 11–14 · roadmap Fase 2
> **Semver:** minor. Aditivo, sem quebra.
> **Risco:** baixo.

## Objetivo

Preparar a API para crescimento: identificar séries por `id`, permitir
atualização dinâmica de opções e séries, e agrupar mutações em lote para um
único render.

## Escopo desta versão

### Identificadores de série (§5.1 · §15.11)

- [ ] Toda série pode ter `id`; IDs duplicados são rejeitados.
- [ ] Métodos aceitam `id` (não só índice).
- [ ] Mensagens de erro exibem o `id`.
- [ ] Exemplos passam a usar `id`, não índice.

### Atualização dinâmica de opções (§5.2 · §15.12)

- [ ] `setOptions` para alterar opções sem recriar o gráfico.
- [ ] Alterar cor não recalcula domínio; alterar eixo recalcula layout.
- [ ] Ocultar série atualiza domínio conforme configuração.

### Adicionar/remover/mostrar/ocultar séries (§15.13)

- [ ] Add/remove de séries em runtime sem vazamento de memória.
- [ ] Show/hide sem recriar o gráfico.

### Operações em lote (§5.3 · §15.14)

- [ ] Uma API de batch agrupa várias mutações.
- [ ] Atualizações agrupadas resultam em um único frame.

## Pré-requisitos técnicos

- Contratos de dados/validação da [v1.3.0](./v1.3.0-contratos-dados.md)
  concluídos (add/remove e batch dependem de validação estável).

## Arquivos provavelmente afetados

- `src/charts/chart-base.ts` — registro de séries por `id`, `setOptions`, batch.
- `src/data/series-store.ts` — indexação por `id`, add/remove.
- `src/types.ts` — `id` em série, tipos de `setOptions` e batch.
- `src/index.ts` — exportar a nova superfície pública.

## Critérios de aceite

- [ ] Toda série pode ter `id`; IDs duplicados são rejeitados.
- [ ] Métodos aceitam `id`; mensagens de erro exibem o `id`; exemplos usam `id`.
- [ ] Alterar cor não recalcula domínio; alterar eixo recalcula layout.
- [ ] Ocultar série atualiza domínio conforme configuração.
- [ ] Adicionar e remover séries não causa vazamento.
- [ ] Atualizações são agrupadas em um único frame.
