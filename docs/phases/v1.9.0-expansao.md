# v1.9.0 — Expansão do núcleo XY (Fase 5)

> **Origem:** improvements §7.1, §7.2, §7.3, §11.3 · §15 passos 25–26 · roadmap Fase 5 · addons §1.4, §3
> **Semver:** minor. Aditivo.
> **Risco:** médio.

## Objetivo

Fechar o núcleo XY com recursos de alto retorno — bar chart, anotações/bandas,
eventos de interação avançados, subpath exports e export CSV — sem perder o foco
em _live and large XY data_.

## Escopo desta versão

### Bar chart (§7.1 · §15.25)

Reaproveita a escala `band` preparada na
[v1.6.0](./v1.6.0-eixo-temporal.md) — não é uma segunda batalha de escala.

- [ ] Vertical e horizontal; agrupado e empilhado; positivo e negativo.
- [ ] Tooltip e crosshair; suporte a viewport; acessibilidade por teclado.
- [ ] Benchmark específico.

### Anotações e bandas (§7.2 · §15.26)

- [ ] Anotações acompanham zoom e pan; labels não extrapolam o canvas.
- [ ] Bandas funcionam em ambos os eixos; ordem de renderização controlável.
- [ ] Atualização dinâmica funciona.

### Eventos de interação avançados (§7.3)

- [ ] Eventos com tipos públicos; listeners removíveis; `destroy()` limpa.
- [ ] Throttling para evitar emissão excessiva.
- [ ] Pointer, teclado e API programática usam o mesmo modelo.

### Subpath exports (§11.3)

- [ ] `package.json` `exports` evoluído para entradas por subpath
      (ex.: `goro-charts/line`, `goro-charts/bar`) mantendo `.` compatível.

### Export CSV do viewport (addons §1.4) — opcional

- [ ] Utilitário que exporta os dados visíveis (SeriesView + viewport) em CSV.
- [ ] Não assume responsabilidade de ingestão (continua da aplicação).

## Pré-requisitos técnicos

- Escala `band` preparada na v1.6.0.
- Viewport da [v1.7.0](./v1.7.0-viewport-zoom-pan.md) (anotações/CSV dependem dele).
- `sideEffects: false` já presente no `package.json` (necessário para subpath tree-shaking).

## Arquivos provavelmente afetados

- `src/charts/bar-chart.ts` (novo) — reaproveitando `band` de `src/math/scale.ts`.
- `src/render/` — render de barras, anotações e bandas.
- `src/charts/chart-base.ts` — eventos avançados, ordem de render.
- `src/index.ts` + novos entry-points — subpath exports.
- `package.json` — `exports` por subpath.
- Utilitário de CSV (novo) — consome `SeriesView` + viewport.

## Critérios de aceite

- [ ] Bar chart: vertical/horizontal, agrupado/empilhado, positivo/negativo.
- [ ] Bar chart: tooltip, crosshair, viewport, acessibilidade por teclado, benchmark.
- [ ] Anotações acompanham zoom/pan; labels não extrapolam; bandas em ambos os eixos.
- [ ] Ordem de renderização controlável; atualização dinâmica funciona.
- [ ] Eventos com tipos públicos, removíveis; `destroy()` limpa; throttling presente.
- [ ] Pointer/teclado/API usam o mesmo modelo de evento.
- [ ] Subpath exports funcionam com tree-shaking; `.` permanece compatível.

## Escopo descartado (addons §3) — não reabrir

| Ideia descartada                              | Motivo                               |
| --------------------------------------------- | ------------------------------------ |
| Sistema de plugins                            | Prematuro; trava o core              |
| Sistema de temas extenso                      | Fora do foco                         |
| Pizza, donut, radar, funil, gauge, mapas      | Não são XY                           |
| "Escala plugável abre qualquer gráfico"       | Serve só p/ time + band dentro de XY |
| Carregamento assíncrono/virtualizado de dados | Responsabilidade da aplicação        |

> OHLC/candlestick permanece como futuro possível (é XY), não descartado.
