# v1.9.0 — Finalização de performance da series layer

Este documento registra o fechamento técnico da fase 1.9.0 de performance. A
implementação final prioriza ganhos medidos com mudanças locais e reversíveis no
pipeline atual, sem introduzir cache de segmentos, `Path2D`, WebGL ou novas APIs
públicas.

## 1. Estado final da 1.9.0

Implementado:

- `dirtyDomain` / `dirtyFrame` / `dirtySeries` / `dirtyOverlay` como flags
  canônicas do pipeline;
- frame layer separada da series layer;
- series layer offscreen condicional para ring/streaming (`maxPoints > 0`);
- cache de ticks/labels compartilhado entre grid e eixos;
- fast path quando o frame pode ser reaproveitado;
- crosshair/overlay fora do repaint de séries;
- crosshair sob cursor parado em streaming é recomputado quando dados deslizam;
- dense line renderizado como envelope vertical por coluna no caminho único de
  produção.

Não implementado nesta versão:

- cache de decimação/segmentos;
- cache de `Path2D`;
- append-only de series layer;
- translação/reprojeção de bitmap;
- WebGL / OffscreenCanvas / Worker.

## 2. Decisão sobre dense line

O gargalo restante em D1/D1-slide/D2 passou a ser a complexidade do path e o
custo de `ctx.stroke()`, não apenas o scan de pontos. Por isso, a otimização que
entrou na 1.9.0 reduz os comandos enviados ao Canvas.

Regime final em `renderLine`:

- sparse: mantém a polilinha real ponto a ponto;
- dense (`nVisible > 2 × plot.w`): cada coluna CSS acumula `minY/maxY` e emite
  uma linha vertical `min→max` no centro da coluna.

Esse desenho preserva o envelope visual da série em janelas densas e reduz a
complexidade para aproximadamente uma linha por coluna com dados.

## 3. Tentativas descartadas

### Cache de decimação em data-space

Foi avaliado como próximo passo natural, mas não entrou no core porque o
profiling indicou que o custo dominante estava em `ctx.stroke()` e na quantidade
de comandos do path. Cachear buckets sem reduzir a complexidade do path não
resolveria o gargalo principal.

Só reabrir essa linha se profiling futuro mostrar que scan/decimation voltou a
dominar. Nesse caso, a unidade correta deve ser coluna/faixa X do renderer, não
um bucket arbitrário de sequência.

### `Path2D`

Adiado. Pode aumentar memória e não há evidência suficiente de ganho consistente
em node-canvas/browser para esta fase. Só implementar com benchmark específico.

### Append-only / translação de bitmap

Adiado. Append-only ingênuo é incorreto quando o domínio X desliza, porque
pixels antigos ficam projetados em posições erradas. Só considerar quando:

- domínio X não mudou; ou
- viewport é fixo; ou
- existir mecanismo explícito de reprojeção/translação que preserve pixels
  antigos corretamente.

## 4. Invariantes preservados

- `setViewport({ xMin, xMax, yAuto? })` continua tendo prioridade sobre domínio
  automático, streaming e grid pinned;
- streaming não limpa viewport; apenas reclampa quando necessário;
- renderers continuam puros e sem acesso direto às stores;
- hot paths mantêm aritmética inline;
- `gapMode` continua respeitado em line/area:
  - `break`: não desenha envelope em colunas somente NaN;
  - `connect`: pula gaps e mantém as colunas com dados;
  - `zero`: trata NaN como `0` durante o draw;
- legend fica na frame layer;
- mudanças visuais de série invalidam series layer e só invalidam frame quando
  afetam legenda visível;
- mudanças estruturais de série ainda invalidam domínio, frame e series.

- legenda fica na frame layer;
- DPR aplica transform nas camadas;
- `destroy()` limpa referências e segue idempotente;
- dense line usa envelope vertical e mantém `gapMode`.

## 6. Benchmarks de aceite

Script:

```txt
node scripts/render-baseline.mjs
```

O script mede o ciclo `appendFrame + draw`/`append + draw` contra
`dist/goro-charts.js`, após `npm run build`.

Cenários:

- D1: `10k × 3 séries`, ring;
- D1-slide: ring cheio, janela deslizando;
- D2: `100k × 3 séries`, ring;
- B1: `20 charts × 2 séries`;
- fast path viewport fixo.

Critérios finais:

- D1 p50 < 16 ms;
- D1-slide p50 < 16 ms;
- D2 p50 < 60 ms, com melhora significativa documentada versus 1.8.0;
- B1 não regredir versus 1.8.0;
- fast path viewport fixo p50 < 2 ms;
- bundle continua dentro do `size-limit`;
- nenhum scaffold experimental (`GORO_DENSE_LINE_MODE`, profiling global ou
  cache incompleto) permanece no bundle.

Resultado final medido em node-canvas 3.2.3, 800×400:

| Cenário                    | p50       | p95       | Status |
| -------------------------- | --------- | --------- | ------ |
| D1 (10k × 3 séries, ring)  | 7.111 ms  | 8.890 ms  | PASS   |
| D1-slide (ring cheio)      | 7.264 ms  | 8.275 ms  | PASS   |
| D2 (100k × 3 séries, ring) | 57.510 ms | 63.055 ms | PASS   |
| B1 (20 charts × 2 séries)  | 22.721 ms | 28.104 ms | PASS   |
| Fast path (viewport fixo)  | 0.621 ms  | 1.217 ms  | PASS   |
